# Sondages v2 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development pour exécuter tâche par tâche (implémenteur + revue entre chaque).

**Goal :** un même sondage WhatsApp natif publiable sur plusieurs surfaces (chaîne, groupe staff, statut-teaser) avec dépouillement des votes dans le web.

**Architecture :** évolution de la verticale `polls` existante (table `polls` + `poll-worker` bot + onglet Sondages web). On remplace le pilotage mono-`target` par un tableau `surfaces`, on stocke l'id du message natif par surface pour relire les votes, et le statut-teaser réutilise la table `statuses` + le status worker déjà déployés. Dépouillement à la demande (pas de nouveau worker).

**Tech Stack :** pnpm/Node 22 ; @goutatou/whapi (client REST) ; @goutatou/service-whatsapp (bot tsx, Railway) ; @goutatou/web (Next 15, Netlify) ; Supabase RLS.

## Global Constraints

- Migration `20260713000027`, `notify pgrst, 'reload schema'` après DDL.
- Sondage natif = message (`sendPoll`/`sendQuiz`) → chaîne/groupe uniquement. **Jamais de vote natif dans un statut** : le `status_teaser` est un STATUT (texte/image) annonçant le sondage + CTA vers la chaîne. **Cocher `status_teaser` force `channel`.**
- Surfaces valides v2 : `channel`, `group`, `status_teaser`. `optin` RETIRÉ de la création (colonne `target` conservée pour compat, valeur neutre `'channel'` à la création v2).
- Best-effort par surface : une surface qui échoue n'empêche pas les autres ; `surface_status` (jsonb) garde le détail.
- Upload fichier (image teaser) = DIRECT navigateur→bucket `status-media` (jamais de Server Action fichier — pattern statuts/chaîne).
- Jamais de prop fonction Server Component → Client Component (crash RSC prod).
- Gating : Sondages = premium (inchangé — vérifier le helper réel dans `@/lib/premium`).
- FR partout, copies figées. `now`/effets injectés côté worker.

## Task SV1 — Migration 0027 + types + readPollResults (whapi)

**Files :**
- Create : `supabase/migrations/20260713000027_sondages_v2.sql`
- Modify : `packages/db/src/types.ts`, `packages/whapi/src/client.ts`, `packages/whapi/test/client.test.ts`

**Migration** (idempotente) :
```sql
alter table polls add column if not exists surfaces text[] not null default '{}';
alter table polls add column if not exists channel_message_id text;
alter table polls add column if not exists group_message_id text;
alter table polls add column if not exists teaser_image_url text;
alter table polls add column if not exists status_id uuid references statuses(id) on delete set null;
alter table polls add column if not exists surface_status jsonb not null default '{}'::jsonb;
update polls set surfaces = array['channel'] where target = 'channel' and surfaces = '{}';
notify pgrst, 'reload schema';
```
Ne PAS appliquer en prod (déploiement = SV5).

**Types** (`packages/db/src/types.ts`) : `export type PollSurface = 'channel' | 'group' | 'status_teaser'`. Si un type Poll explicite existe, ajouter les champs ; sinon ne rien inventer.

**whapi — `readPollResults`** : AJOUTER une méthode (NE PAS modifier `readPollVotes`, utilisée par les decision workers) :
```
async readPollResults(messageId: string): Promise<{ options: Array<{ label: string; count: number }>; total: number }>
```
Même lecture que `readPollVotes` (GET /messages/{id}, `message.poll.results`, `count`/`votes`/`voters.length`) mais renvoie le décompte par option (label = `name`||`title`) + total. Test whapi : parse d'un poll.results à 3 options → counts corrects + total.

**Tests** : build packages verts + le test whapi ci-dessus.

## Task SV2 — Bot : poll-worker multi-surfaces

**Files :**
- Modify : `services/whatsapp/src/polls/repo.ts`, `services/whatsapp/src/polls/worker.ts`
- Test : `services/whatsapp/test/poll-worker.test.ts` (+ repo test si présent)

**repo.ts** :
- `ClaimedPoll` gagne `surfaces: PollSurface[]`, `teaserImageUrl: string | null` ; garder `target` (compat). `claimQueued` select + map incluent `surfaces, teaser_image_url`.
- `PollChannel` gagne `waChannelInvite: string | null` (lien d'invitation chaîne pour le CTA teaser — `restaurants.wa_channel_invite`) et `staffGroupId: string | null` (`restaurants.staff_group_id`). Étendre le select `getChannel`.
- Nouvelles méthodes :
  - `recordSurface(pollId, surface, patch)` : met à jour `surface_status` (merge jsonb : `surface_status || '{"<surface>":"sent"}'`), et selon la surface `channel_message_id`/`group_message_id`. (Implémentation : lire surface_status courant puis update, OU update jsonb par `||`.)
  - `insertTeaserStatus(restaurantId, content, mediaUrl | null)` : insère dans `statuses` (kind = mediaUrl ? 'image' : 'text', state 'scheduled', scheduled_at = now, content, media_url, audience 'all', auto_generated false) → renvoie l'id inséré ; puis `update polls set status_id=<id>`.
  - `finishMulti(pollId, { status, sentCount, error })` : comme `finish` existant (réutilisable tel quel).

**worker.ts** — `processPollOnce` réécrit pour parcourir `poll.surfaces` (fallback : si `surfaces` vide mais `target==='channel'`, traiter comme `['channel']` — non-régression lignes historiques ; `target==='optin'` conservé tel quel pour les lignes historiques uniquement, non produit par la v2) :
- `makeWhapi` type étendu : `Pick<WhapiClient, 'sendPoll' | 'sendQuiz' | 'postStatusText' | 'postStatusMedia'>`.
- Pour chaque surface cochée, best-effort (try/catch, log, `recordSurface(...,'failed')` sur échec) :
  - `channel` : canal + `waChannelId` requis → `send(waChannelId)` (send = quiz si quizCorrect non nul sinon poll) → `recordSurface(pollId, 'channel', { status:'sent', messageId: res.id })`.
  - `group` : `staffGroupId` requis (sinon failed) → `send(staffGroupId)` → `recordSurface(pollId,'group',{status:'sent', messageId:res.id})`.
  - `status_teaser` : `waChannelId` + `waChannelInvite` requis (sinon failed — rien à annoncer) → texte `📊 ${question}\n\nVotez sur notre chaîne 👉 ${waChannelInvite}` → `insertTeaserStatus(restaurantId, texte, teaserImageUrl)` (le status worker publiera) → `recordSurface(pollId,'status_teaser',{status:'sent'})`.
- Statut global : `sent` si ≥1 surface sent, sinon `failed`. `sentCount` = nb surfaces sent. Best-effort global conservé (le try/catch externe reste).

**Tests** (`poll-worker.test.ts`, mocks repo + makeWhapi) :
- surfaces=['channel'] → sendPoll(waChannelId) + recordSurface channel sent + channel_message_id ;
- surfaces=['group'] → sendPoll(staffGroupId) ; groupe absent → failed ;
- surfaces=['status_teaser'] avec chaîne+invite → insertTeaserStatus appelé avec texte contenant le lien ; sans invite → failed ;
- surfaces=['channel','group','status_teaser'] : les 3 tentées, une qui échoue → les autres OK, surface_status détaillé ;
- quizCorrect non nul → sendQuiz ;
- rétrocompat : ligne surfaces=[] + target='channel' → traitée comme channel.
